# Weak Authentication Walkthrough

This walkthrough explains the password storage demo in the local vulnerable web app.

## Scope

This demo must be run locally only:

```text
http://127.0.0.1:5000
```

Do not deploy intentionally weak authentication code to a public network.

## Weak Behavior

The weak authentication table stores passwords as plaintext:

```text
username: carol
password: letmein123
```

The weak login route retrieves the stored password and compares it directly with the submitted password.

## Impact

If a database containing plaintext passwords is exposed, every stored password is immediately exposed. Users often reuse passwords, so a leak can also affect accounts on other services.

## Remediation

The secure authentication table stores password hashes generated by Werkzeug:

```python
generate_password_hash("letmein123")
```

The secure login route verifies the submitted password with:

```python
check_password_hash(user["password_hash"], password)
```

The application never needs to store or recover the original password.

## Key Lesson

Passwords should not be stored as plaintext. Store salted password hashes and verify login attempts with a password hashing function designed for authentication.
