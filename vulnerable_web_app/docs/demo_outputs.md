# Demo Outputs

Example outputs captured from the local demo app. Each section shows the intentionally vulnerable behavior and the secure behavior side by side.

---

## SQL Injection

**Payload**

```text
username: ' OR '1'='1' --
password: anything
```

**Vulnerable route** `/vulnerable-login`

The input is formatted directly into the query string. The injected payload changes the query logic so the `AND password` condition is commented out and the `OR '1'='1'` condition always evaluates to true.

```sql
SELECT id, username FROM users WHERE username = '' OR '1'='1' --' AND password = 'anything'
```

Result: `Login successful. Welcome, alice.`

**Secure route** `/secure-login`

The parameterized query keeps the input as data. The same payload is treated as a literal username string and finds no matching user.

```sql
SELECT id, username FROM users WHERE username = ? AND password = ?
```

Result: `Login failed.`

## Cross-Site Scripting

**Payload**

```html
<script>alert('xss')</script>
```

**Vulnerable route** `/vulnerable-comment`

The template renders input with `{{ comment|safe }}`. The raw script tag appears in the response and executes in the browser.

```
<script>alert('xss')</script>
```

**Secure route** `/secure-comment`

The template renders input with `{{ comment }}`. Jinja's default escaping converts the script tag into plain text that the browser displays instead of executing.

```
&lt;script&gt;alert(&#39;xss&#39;)&lt;/script&gt;
```

## Password Storage

**Account**

```text
username: carol
password: letmein123
```

**Vulnerable route** `/weak-auth-login`

The password is stored as plaintext. The result page shows the raw stored value.

```
Stored password: letmein123
```

**Secure route** `/secure-auth-login`

The password is stored as a salted hash generated by Werkzeug. The result page shows the hash. The original password cannot be recovered from this value.

```
Stored password hash: scrypt:32768:8:1$F05t87IqK354WjVw$33da8e6df5845762ae8bf7bd22dbf3c5298770b9485847a001147805a8e8374c3d135c443b4707e363b2cc715d581516cd6a7021917118c3fd00f99408db2b71
```

The exact hash value can differ between database initializations because password hashing includes a random salt.
